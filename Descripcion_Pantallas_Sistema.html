Descripción de Pantallas del Sistema

Sistema de Ofertas Laborales para Tacna

Documentación técnica de interfaces y funcionalidades

1. Pantalla de Inicio de Sesión (Login)

Autenticación de usuarios

Ruta de Acceso
GET /login
POST /login
Requisitos de Acceso
Pública (sin autenticación)
Descripción General

Pantalla inicial del sistema que permite a los usuarios autenticarse mediante credenciales (usuario y contraseña). Es el punto de entrada principal y redirige automáticamente a usuarios ya autenticados al dashboard. Incluye validación de credenciales y mensajes de error informativos.

Funcionalidades Principales
Autenticación de Usuarios: Permite ingresar con usuario y contraseña. Valida credenciales contra la base de datos MongoDB o permite acceso en modo offline con credenciales predefinidas (admin/admin123).
Validación de Formulario: Campos requeridos (usuario y contraseña) con validación HTML5 y retroalimentación visual.
Mensajes de Estado: Muestra mensajes flash informativos sobre el resultado del intento de login (éxito, error, advertencias sobre modo offline).
Redirección Automática: Si el usuario ya tiene sesión activa, es redirigido automáticamente al dashboard. Si no tiene sesión, permanece en la pantalla de login.
Modo Offline: Soporta funcionamiento sin conexión a MongoDB, permitiendo login con credenciales predefinidas y mostrando advertencia correspondiente.
Elementos de Interfaz
  • Header: Muestra el título "Sistema de Ofertas - Tacna"
  • Formulario de login: Campos de usuario y contraseña
  • Botón de inicio de sesión: Botón principal para autenticarse
  • Información de credenciales: Muestra las credenciales por defecto (admin/admin123)
  • Alertas flash: Mensajes de éxito, error o advertencia según el resultado

2. Dashboard (Panel Principal)

Vista general del sistema

Ruta de Acceso
GET /dashboard
GET /
Requisitos de Acceso
Requiere autenticación
Descripción General

Pantalla principal del sistema que proporciona una vista consolidada de las ofertas laborales y estadísticas clave. Es la primera pantalla que ven los usuarios después de iniciar sesión. Muestra métricas generales, ofertas recientes y acceso rápido a las principales funcionalidades.

Funcionalidades Principales
Tarjetas de Estadísticas: Muestra 4 tarjetas con métricas clave: Total de ofertas, ofertas de Computrabajo, ofertas de Bumeran, y ofertas de Indeed. Cada tarjeta tiene un color distintivo y un ícono representativo.
Lista de Ofertas Recientes: Presenta las 10 ofertas más recientes en formato de tabla, mostrando título, empresa, nivel académico, modalidad, salario, fuente y fecha de publicación.
Acceso Rápido a Funciones: Botón "Extraer Ofertas" que permite ejecutar el proceso de scraping desde el dashboard.
Navegación Integrada: Enlaces rápidos a otras secciones del sistema (Ofertas, Estadísticas) desde la barra de navegación.
Datos de Simulación: Si la base de datos está vacía o no hay conexión, muestra datos mock para demostración.
Elementos de Interfaz
  • Header con título: "Dashboard" con ícono de tacómetro y botón de extracción
  • Tarjetas de métricas: 4 tarjetas con colores (azul, verde, naranja, rojo) mostrando estadísticas
  • Tabla de ofertas recientes: Tabla responsive con información resumida de las ofertas
  • Badges de estado: Indicadores visuales para nivel académico y modalidad de trabajo
  • Enlaces de acción: Botones para ver detalles y enlaces externos a ofertas originales

3. Lista de Ofertas Laborales

Búsqueda y filtrado de ofertas

Ruta de Acceso
GET /ofertas
Requisitos de Acceso
Requiere autenticación
Descripción General

Pantalla principal para explorar y buscar ofertas laborales. Permite filtrar ofertas por múltiples criterios (búsqueda de texto, nivel académico, modalidad, empresa) y visualizar los resultados en dos formatos diferentes: tabla o tarjetas. Incluye paginación para manejar grandes volúmenes de datos.

Funcionalidades Principales
Búsqueda de Texto: Campo de búsqueda que permite buscar en título de oferta, puesto y nombre de empresa. La búsqueda es case-insensitive y utiliza expresiones regulares.
check-circle me-2 text-success">Filtros Avanzados:
  • Nivel Académico: Filtro por Practicante, Bachiller o Profesional
  • Modalidad: Filtro por Presencial, Remoto o Híbrido
  • Empresa: Búsqueda específica por nombre de empresa
check-circle me-2 text-success">Vista Dual: Permite alternar entre vista de tabla (compacta, ideal para comparar) y vista de tarjetas (visual, ideal para explorar). El usuario puede cambiar entre ambas vistas con botones dedicados.
check-circle me-2 text-success">Paginación: Muestra 20 ofertas por página con navegación entre páginas. Los filtros se mantienen al cambiar de página.
check-circle me-2 text-success">Acciones por Oferta: Cada oferta tiene botones para ver detalles completos y abrir la oferta original en el portal de origen.
check-circle me-2 text-success">Extracción de Ofertas: Botón para ejecutar el proceso de scraping y actualizar la base de datos con nuevas ofertas.
check-circle me-2 text-success">Contador de Resultados: Muestra el número total de ofertas encontradas según los filtros aplicados.
Elementos de Interfaz
  • Header con acciones: Título "Ofertas Laborales" con botones de extracción y filtros
  • Formulario de filtros: Card con campos de búsqueda y selectores de filtros
  • Vista de tabla: Tabla responsive con columnas: Título, Empresa, Nivel, Experiencia, Modalidad, Salario, Fuente, Fecha, Acciones
  • Vista de tarjetas: Grid de tarjetas con información resumida y visualmente atractiva
  • Selector de vista: Botones para alternar entre vista tabla y tarjetas
  • Mensaje de sin resultados: Muestra mensaje informativo cuando no hay ofertas que coincidan con los filtros

4. Detalle de Oferta Laboral

Información completa de una oferta

Ruta de Acceso
GET /ofertas/<oferta_id>
Requisitos de Acceso
Requiere autenticación
Descripción General

Pantalla de detalle que muestra toda la información disponible de una oferta laboral específica. Incluye información completa sobre el puesto, empresa, requisitos, responsabilidades, conocimientos necesarios y documentos requeridos. Proporciona enlaces para acceder a la oferta original en el portal.

Funcionalidades Principales
check-circle me-2 text-success">Información Completa: Muestra todos los detalles de la oferta: título, empresa, puesto, ubicación, modalidad, experiencia requerida, jornada, salario, nivel académico.
check-circle me-2 text-success">Responsabilidades: Sección dedicada que describe las responsabilidades principales del puesto de trabajo.
check-circle me-2 text-success">Conocimientos Clave: Lista de conocimientos, habilidades o tecnologías requeridas, presentadas como badges para fácil visualización.
check-circle me-2 text-success">Documentos Requeridos: Lista de documentos que el candidato debe presentar para postular al puesto.
check-circle me-2 text-success">Enlace a Oferta Original: Botón que abre la oferta en el portal de origen (Computrabajo, Bumeran, etc.) en una nueva pestaña.
check-circle me-2 text-success">Navegación con Breadcrumbs: Ruta de navegación (breadcrumbs) que permite volver fácilmente al dashboard o a la lista de ofertas.
check-circle me-2 text-success">Información de Fuente: Muestra de qué portal fue extraída la oferta y la fecha de publicación original.
Elementos de Interfaz
  • Breadcrumbs: Navegación Dashboard > Ofertas > [Título de Oferta]
  • Header de oferta: Título grande, nombre de empresa y badge de nivel académico
  • Información en dos columnas: Layout organizado que distribuye la información principal
  • Secciones expandidas: Responsabilidades, conocimientos clave y documentos en secciones separadas
  • Badges de conocimientos: Etiquetas visuales para cada conocimiento requerido
  • Botón de acción principal: Botón destacado para ver la oferta original
  • Información adicional: Fecha de publicación, fuente, URL de la oferta

5. Estadísticas y Análisis

Análisis de datos de ofertas

Ruta de Acceso
GET /estadisticas
Requisitos de Acceso
Requiere autenticación
Descripción General

Pantalla de análisis que presenta estadísticas detalladas sobre las ofertas laborales almacenadas en el sistema. Utiliza gráficos interactivos (Chart.js) para visualizar distribuciones por nivel académico, modalidad, fuente y empresas más activas. Proporciona insights valiosos para entender el mercado laboral en Tacna.

Funcionalidades Principales
check-circle me-2 text-success">Tarjetas de Resumen: Cuatro tarjetas con métricas clave: Total de ofertas, ofertas para Profesionales, Técnicos y Bachilleres. Cada tarjeta tiene un color distintivo y un ícono representativo.
check-circle me-2 text-success">Gráfico de Distribución por Nivel: Gráfico de pastel (pie chart) que muestra la proporción de ofertas según el nivel académico requerido (Profesional, Bachiller, Practicante).
check-circle me-2 text-success">Gráfico de Distribución por Modalidad: Gráfico de dona (doughnut chart) que visualiza la distribución de ofertas según la modalidad de trabajo (Presencial, Remoto, Híbrido).
check-circle me-2 text-success">Gráfico de Distribución por Fuente: Gráfico de barras horizontales que muestra cuántas ofertas provienen de cada portal (Computrabajo, Bumeran, Indeed, Trabajos.pe).
check-circle me-2 text-success">Tabla de Top Empresas: Tabla que lista las 10 empresas que más ofertas han publicado, ordenadas por cantidad de ofertas.
check-circle me-2 text-success">Información del Sistema: Sección que muestra la última actualización, número de fuentes monitoreadas y estado del sistema.
Datos de Simulación: Si la base de datos está vacía, muestra estadísticas calculadas a partir de datos mock para demostración.
Elementos de Interfaz
  • Header descriptivo: Título "Estadísticas" con descripción del propósito
  • Tarjetas de métricas: 4 tarjetas con colores (azul, verde, azul claro, amarillo) y grandes números
  • Gráficos interactivos: Canvas con Chart.js para visualización de datos
  • Tabla de empresas: Tabla responsive con ranking de empresas más activas
  • Card de información del sistema: Información técnica sobre el estado y configuración
  • Fecha de actualización dinámica: Muestra la fecha y hora actual usando JavaScript

6. Funcionalidad de Extracción de Ofertas

Proceso de scraping automático

Ruta de Acceso
POST /extraer
Requisitos de Acceso
Requiere autenticación
Descripción General

Funcionalidad backend que ejecuta el proceso de web scraping para extraer ofertas laborales de múltiples portales (Computrabajo, Bumeran, Indeed, Trabajos.pe). Se ejecuta mediante una petición AJAX desde cualquier pantalla que tenga el botón "Extraer Ofertas". El proceso es asíncrono y muestra notificaciones de progreso y resultado.

Funcionalidades Principales
check-circle me-2 text-success">Scraping Multi-Portal: Extrae ofertas de 4 portales diferentes: Computrabajo, Bumeran, Indeed y Trabajos.pe, utilizando técnicas de web scraping con BeautifulSoup y Requests.
check-circle me-2 text-success">Procesamiento Asíncrono: Se ejecuta en segundo plano sin bloquear la interfaz del usuario. Muestra indicadores de carga durante el proceso.
check-circle me-2 text-success">Deduplicación: Evita insertar ofertas duplicadas comparando el ID único de cada oferta antes de guardarla.
check-circle me-2 text-success">Notificaciones: Muestra mensajes de éxito o error al finalizar el proceso, indicando cuántas ofertas nuevas se agregaron o si hubo algún problema.
check-circle me-2 text-success">Manejo de Errores: Captura y registra errores de conexión, parsing o base de datos, proporcionando mensajes informativos al usuario.
check-circle me-2 text-success">Actualización Automática: Después de una extracción exitosa, el usuario puede ver las nuevas ofertas en las pantallas de Dashboard y Lista de Ofertas.
Elementos de Interfaz
  • Botón "Extraer Ofertas": Disponible en Dashboard y Lista de Ofertas, con ícono de sincronización
  • Indicador de carga: Muestra estado de carga durante el proceso (mediante JavaScript)
  • Alertas de resultado: Mensajes flash que informan sobre el éxito o fallo de la extracción
  • Respuesta JSON: El endpoint retorna información estructurada sobre el resultado del proceso

Flujo de Navegación del Sistema

Flujo Principal:
  1. Login → Autenticación del usuario
  2. Dashboard → Vista general y ofertas recientes
  3. Ofertas → Búsqueda y filtrado de ofertas
  4. Detalle de Oferta → Información completa de una oferta específica
  5. Estadísticas → Análisis y gráficos de datos
Accesos Rápidos:
  • Desde cualquier pantalla: Barra de navegación para cambiar de sección
  • Desde Dashboard/Ofertas: Botón "Extraer Ofertas" para actualizar datos
  • Desde Lista de Ofertas: Botón "Ver detalles" para ver oferta completa
  • Desde cualquier pantalla: Menú de usuario para cerrar sesión

Tecnologías y Herramientas

Frontend:
  • HTML5
  • CSS3 (Bootstrap 5.1.3)
  • JavaScript (Vanilla)
  • Chart.js (gráficos)
  • Font Awesome 6.0 (íconos)
  • Jinja2 (templating)
Backend:
  • Flask (framework web)
  • Python 3.x
  • PyMongo (MongoDB driver)
  • BeautifulSoup4 (scraping)
  • Requests (HTTP client)
  • Werkzeug (seguridad)
Base de Datos:
  • MongoDB (NoSQL)
  • Datos de simulación (mock data)
  • Soporte offline

Recomendaciones

Mejoras Funcionales:
  • Implementar sistema de notificaciones por correo electrónico para nuevas ofertas que coincidan con perfiles de búsqueda guardados.
  • Agregar funcionalidad de favoritos para que los usuarios puedan guardar ofertas de interés.
  • Desarrollar un módulo de reportes exportables en formato PDF o Excel para análisis externos.
  • Implementar búsqueda avanzada con múltiples criterios combinados y operadores lógicos (AND, OR, NOT).
  • Agregar sistema de calificación o comentarios para las ofertas por parte de los usuarios.
Mejoras Técnicas:
  • Implementar caché de resultados de búsqueda para mejorar el rendimiento del sistema.
  • Agregar pruebas unitarias y de integración para garantizar la calidad del código.
  • Implementar logging más detallado para facilitar el diagnóstico de problemas en producción.
  • Desarrollar una API REST completa para permitir la integración con otros sistemas.
  • Implementar paginación más eficiente utilizando cursores en lugar de offsets para grandes volúmenes de datos.
Mejoras de Seguridad:
  • Implementar autenticación de dos factores (2FA) para mayor seguridad en el acceso.
  • Agregar rate limiting en las rutas de extracción para prevenir abuso del sistema.
  • Implementar validación más estricta de inputs para prevenir inyecciones y ataques XSS.
  • Agregar cifrado de datos sensibles en la base de datos.
  • Implementar auditoría de accesos y cambios en el sistema.
Mejoras de Usabilidad:
  • Implementar diseño responsive completo para dispositivos móviles y tablets.
  • Agregar tutorial interactivo para nuevos usuarios del sistema.
  • Implementar búsquedas guardadas con alertas automáticas.
  • Agregar modo oscuro como opción de visualización.
  • Implementar ayuda contextual en cada pantalla para facilitar el uso del sistema.
Mejoras de Mantenibilidad:
  • Documentar todos los módulos y funciones con docstrings completos siguiendo estándares Python.
  • Implementar CI/CD para automatizar pruebas y despliegues.
  • Crear documentación técnica más detallada para desarrolladores.
  • Establecer un proceso de versionado semántico para el sistema.
  • Implementar monitoreo y alertas para el estado del sistema en producción.
